## 1. プロジェクト概要

日米株式（日経225 + S&P500）を対象とした中期スイング戦略の自動化システム。  
元本 300万円、楽天証券で手動発注、シグナル生成はシステムが担う。

**4シナリオ構成**

| ID | 戦略名 | 概要 |
|----|--------|------|
| S2 | 52週高値ブレイクアウト | 年初来高値更新＋出来高急増でエントリー |
| S3 | 押し目買い | RSI売られすぎ→回復＋上昇トレンド |
| S4 | 決算後ドリフト（PEAD） | 決算サプライズ翌々日にエントリー |
| S6 | 平均回帰（短期反発） | 急落後の短期反発を狙う |

## 6. 積み残し課題

### 🔴 最重要：バックテスト成績が実弾投入基準を大幅未達

現時点でPhase 5（ペーパートレード）に進んでも、3ヶ月後に基準クリアできる見通しがない。  
**先にバックテスト成績を改善する方針か、現成績でもペーパー開始するかの判断が必要。**

原因仮説（要検証）：
1. **パラメータ未最適化**：現在は scenarios.md の推奨値をそのまま使用。グリッドサーチ候補を広げれば改善余地あり
2. **決算データ不足**：S4（PEAD）が EPS データなしでギャップ＋出来高のみで動作中。本来の強みを発揮できていない
3. **全銘柄適用の弊害**：日経225＋S&P500全726銘柄に適用すると、低品質なシグナルが混入して全体成績を押し下げている可能性
4. **相場環境との非適合**：2021年後半〜2022年（金利急騰局面）でパフォーマンスが特に悪化している

## 8. 方針検討の論点

### 論点A：バックテスト成績改善を先行させるか？

**選択肢1: パラメータ再チューニングを先行**
- ウォークフォワードのグリッドサーチ候補を拡張して再実行
- 銘柄ユニバースを絞る（例：流動性上位100銘柄に限定）
- S4の決算データを実際に取得してから評価（J-Quantsまたは手動収集）
- メリット：ペーパートレードを意味あるものにできる
- デメリット：チューニングに時間がかかる。過学習リスクもある

**選択肢2: 現状のままPhase 5（ペーパートレード）を開始**
- バックテスト成績が悪くてもペーパーで実際の動作を観察する
- 実運用と乖離している原因（データ品質、フィルタ動作等）がペーパーで明らかになる可能性
- メリット：すぐに動き出せる。実環境での挙動データが得られる
- デメリット：「基準未達で3ヶ月過ごす」ことになる可能性が高い

**選択肢3: 戦略設計を見直す**
- 4シナリオの組み合わせ自体を再検討
- 相場環境フィルタの強化（金利環境、セクターローテーション等）
- より単純な1〜2シナリオに絞ってロバスト性を高める
- メリット：根本的な改善につながる可能性
- デメリット：開発工数が大きい

### 論点B：決算データ（S4シナリオ）をどう確保するか？

- **yfinance**：EPS予想が部分的にしか取れない（特に日本株は困難）
- **J-Quants**：無料枠あり。日本株決算は充実。要アカウント登録
- **手動管理**：主要銘柄のみCSVで管理（現実的な妥協点）
- S4を無効化して残り3シナリオで運用する選択肢もある

### 論点C：ユニバースを絞るべきか？

フルバックテスト（726銘柄）よりパイロット（10銘柄）のほうがシャープが高い（0.20 vs 0.31）。  
ただしパイロットの銘柄選定に選択バイアスが含まれている可能性がある。  
流動性・時価総額フィルタで上位100〜200銘柄に絞ることで成績が改善するか検証する価値はある。
