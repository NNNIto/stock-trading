## 1. 全体方針

### 1.1 共通設定（全シナリオ共通）

**ユニバース**
- 日本株：日経平均225構成銘柄
- 米国株：S&P500構成銘柄
- 合計：約725銘柄

**流動性フィルタ（全シナリオ適用）**
- 過去20営業日平均出来高 ≥ 100,000株（日本株）／500,000株（米国株）
- 過去20営業日平均売買代金 ≥ 5億円（日本株）／1,000万USD（米国株）
- 上記未満の銘柄はシグナル対象外

**ポジションサイズ（MVP段階：案1 統一ルール）**
- 1ポジション = 元本の15%（固定）
- 同時保有上限：7銘柄
- 1銘柄あたり最大投資比率：20%
- 将来段階で案2（シナリオごと個別）→ 案3（ATRベース動的）へ進化予定

**取引コスト（バックテスト反映必須）**
- 株式売買手数料：往復0.1%（楽天証券のゼロコース前提でも、保守的に計上）
- スリッページ：0.2%
- 米国株為替コスト：片道0.5%（楽天証券スプレッド、往復で実質1.0%）
- 税金：譲渡益課税20.315%（バックテスト評価時に税引後リターンも併記）

**約定前提**
- シグナル生成は当日終値ベース
- エントリー約定価格：翌営業日始値
- エグジット約定価格：翌営業日始値（または同日終値、シナリオごとに明記）

## 2. シナリオ S2: 52週高値ブレイクアウト

### 2.1 概要
- **タイプ:** 順張り・トレンドフォロー
- **想定保有期間:** 1〜6ヶ月
- **理論的根拠:** 新高値更新銘柄は継続して上昇しやすい（モメンタム効果）。ミネルヴィニ手法、ターtle Trading等の簡易版

### 2.2 エントリー条件（推奨パラメータ）

以下を全て満たす場合、翌営業日始値で買いエントリー。

1. 終値が過去**252営業日（約52週）**の最高値を更新
2. 当日出来高が過去20営業日平均出来高の**1.5倍以上**
3. 200日移動平均線が**上向き**（過去20日比でプラス）
4. 終値が200日移動平均線より**上**

### 2.3 エグジット条件

以下のいずれかに該当時、翌営業日始値で売却。

- **ロスカット:** エントリー価格から**-8%下落**（終値ベース）
- **トレーリングストップ:** エントリー後の最高値から**-15%下落**
- **トレンド転換:** 終値が20日移動平均線を下抜け、かつ20日MAが下向き
- **時間切れ:** エントリー後**180営業日**経過

### 2.4 パラメータ候補（バックテストで比較）

| パラメータ | 推奨 | 候補 |
|----------|------|------|
| 高値判定期間 | 252日 | 126日（半年高値）、504日（2年高値） |
| 出来高倍率 | 1.5倍 | 1.2倍、2.0倍 |
| ロスカット | -8% | -5%、-10% |
| トレーリング | -15% | -10%、-20% |
| 時間切れ | 180日 | 90日、365日 |

## 3. シナリオ S3: 押し目買い（RSI＋トレンド）

### 3.1 概要
- **タイプ:** 順張り・押し目
- **想定保有期間:** 数週間〜1ヶ月
- **理論的根拠:** 上昇トレンド中の一時的調整局面は買い場になりやすい。RSIで売られすぎを検知し、トレンドフィルタで偽シグナルを除外

### 3.2 エントリー条件（推奨パラメータ）

以下を全て満たす場合、翌営業日始値で買いエントリー。

1. 終値が**50日移動平均線より上**（上昇トレンド継続中）
2. 50日移動平均線が**上向き**（過去10日比でプラス）
3. **RSI(14)** が直近5営業日内に**35以下**まで低下した後、当日**40以上に回復**
4. 当日終値が前日終値より**高い**（反発確認）

### 3.3 エグジット条件

- **利確:** **RSI(14) > 70** に到達、または**+15%上昇**（いずれか早い方）
- **ロスカット:** エントリー価格から**-7%下落**
- **トレンド転換:** 終値が50日移動平均線を下抜け
- **時間切れ:** エントリー後**45営業日**経過

### 3.4 パラメータ候補

| パラメータ | 推奨 | 候補 |
|----------|------|------|
| トレンドフィルタMA | 50日 | 25日、75日、200日 |
| RSI期間 | 14 | 9、21 |
| RSI低下閾値 | 35 | 30、40 |
| RSI回復閾値 | 40 | 45、50 |
| 利確閾値 | RSI>70 or +15% | RSI>65、+10% |
| ロスカット | -7% | -5%、-10% |
| 時間切れ | 45日 | 30日、60日 |

## 4. シナリオ S4: 決算後ドリフト（PEAD）

### 4.1 概要
- **タイプ:** イベントドリブン
- **想定保有期間:** 2〜3ヶ月
- **理論的根拠:** ポジティブサプライズ決算後、株価は数週間〜数ヶ月にわたって上昇継続する傾向（Bernard & Thomas, 1989）。学術的に最も検証されている異常リターン現象の一つ

### 4.2 エントリー条件（推奨パラメータ）

決算発表後、以下を全て満たす場合、決算翌々営業日始値で買いエントリー。

1. 決算発表日に**ギャップアップ**（始値が前日終値より+2%以上）
2. 決算発表日終値が前日終値より**+5%以上上昇**
3. 決算発表日の出来高が過去20営業日平均の**3倍以上**
4. EPS（一株利益）が**事前予想を上回る**（ポジティブサプライズ）
5. 200日移動平均線が上向き（長期トレンドが死んでいない）

### 4.3 エグジット条件

- **時間切れ:** エントリー後**60営業日**経過（PEAD効果の継続期間目安）
- **ロスカット:** エントリー価格から**-10%下落**
- **利確:** **+25%上昇**で半分利確、残りはトレーリングストップ（最高値から-15%）
- **次回決算前:** 次回決算発表5営業日前に強制クローズ

### 4.4 パラメータ候補

| パラメータ | 推奨 | 候補 |
|----------|------|------|
| ギャップアップ閾値 | +2% | +1%、+3% |
| 決算日上昇率閾値 | +5% | +3%、+7% |
| 出来高倍率 | 3倍 | 2倍、5倍 |
| 保有期間 | 60日 | 30日、90日 |
| ロスカット | -10% | -7%、-15% |
| 利確 | +25% | +15%、+30% |

### 4.5 実装上の注意

- 決算データソースは別途検討必要（無料源と有料源の比較を実装フェーズで実施）
- EPS予想データの取得が難しい場合、代替として「ギャップアップ＋出来高急増」のみでエントリー条件を簡易化する選択肢あり

## 5. シナリオ S6: 平均回帰（短期反発）

### 5.1 概要
- **タイプ:** 逆張り・短期
- **想定保有期間:** 数日〜2週間
- **理論的根拠:** 短期的な過剰売りは反発しやすい。ただし長期トレンドが死んでいる銘柄は反発せずそのまま下落するため、トレンドフィルタで除外

### 5.2 エントリー条件（推奨パラメータ）

以下を全て満たす場合、翌営業日始値で買いエントリー。

1. 過去**5営業日リターン**が **-10%以下**
2. 終値が**200日移動平均線より上**（長期トレンドは死んでいない）
3. 当日出来高が過去20営業日平均の**2倍以上**（売りクライマックス確認）
4. RSI(2) < 10（極端な売られすぎ、短期RSI使用）

### 5.3 エグジット条件

- **時間切れ:** エントリー後**10営業日**経過（短期想定）
- **利確:** **+5%上昇** または **RSI(2) > 70**
- **ロスカット:** エントリー価格から**-5%下落**

### 5.4 パラメータ候補

| パラメータ | 推奨 | 候補 |
|----------|------|------|
| 下落判定期間 | 5日 | 3日、10日 |
| 下落率閾値 | -10% | -7%、-15% |
| トレンドフィルタ | 200日MA | 100日MA、150日MA |
| 出来高倍率 | 2倍 | 1.5倍、3倍 |
| 短期RSI期間 | 2 | 3、5 |
| 利確 | +5% or RSI(2)>70 | +3%、+7% |
| ロスカット | -5% | -3%、-7% |
| 時間切れ | 10日 | 5日、15日 |

## 6. シナリオ間の競合解決ルール

### 6.1 同一銘柄に複数シグナル
- 既ポジション銘柄に別シナリオがシグナルを出しても、重ねて建てない
- 同日に複数シナリオが同一銘柄にシグナルを出した場合、シナリオ優先順位は **S4 > S2 > S3 > S6**（イベント＞長期＞中期＞短期）

### 6.2 ポートフォリオ上限到達時
- 同時保有銘柄数が7に達している場合、新規シグナルは見送り
- 例外：S4（PEAD）シグナルは優先度高、既存ポジションの中で**S6保有銘柄を強制クローズして枠を空ける**

### 6.3 セクター集中回避
- 同一セクターの保有銘柄数が3を超える場合、新規シグナルは見送り

## 7. バックテスト評価方法

### 7.1 評価指標（シナリオごとに算出）
- 累積リターン、CAGR
- シャープレシオ、ソルティノレシオ
- 最大ドローダウン、平均ドローダウン
- 勝率、ペイオフレシオ（平均利益/平均損失）
- プロフィットファクター
- 平均保有日数、取引回数
- 対ベンチマーク超過リターン（日経平均、S&P500、バイ＆ホールド）

### 7.2 統計的検定
- 取引回数 < 30 のシナリオは統計的信頼性不足として棄却
- ブートストラップ法でシャープレシオの信頼区間を算出

### 7.3 ウォークフォワード設定
- 学習窓：12ヶ月、検証窓：3ヶ月、ステップ：3ヶ月
- 学習窓内でパラメータ最適化、検証窓で評価
- アウトオブサンプル期間（直近1年）は最終評価のみに使用

## 8. パラメータ変更管理

### 8.1 ファイル形式
- パラメータはYAMLで管理：`config/scenarios/s2.yaml`, `s3.yaml`, `s4.yaml`, `s6.yaml`
- 各YAMLには「現行値」「過去変更履歴」「変更理由」「バックテスト結果サマリー」を含む

### 8.2 変更プロセス
1. パラメータ変更の仮説を立てる
2. バックテストで新旧比較
3. アウトオブサンプル期間でも検証
4. 改善が確認できた場合のみ採用
5. Gitコミット（メッセージに変更理由・改善幅を明記）

### 8.3 変更頻度
- 原則として四半期に1回
- 例外：明確なロジック誤りが発見された場合は随時修正
