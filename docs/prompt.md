### 2. 出口戦略（損切り・利確）の基準値変更

リスクを抑える方向で以下のパラメータを変更する。
**変更前後の対応表：**

|パラメータ            |変更前 |変更後 |
|-----------------|----|----|
|固定損切り（全戦略共通）     |-10%|-7% |
|トレイリングストップ（全戦略共通）|-15%|-10%|
|利益確定（S4戦略）       |+15%|+12%|
|最大保有日数 S2        |180日|120日|
|最大保有日数 S4        |60日 |45日 |
|最大保有日数 S6        |10日 |7日  |

#### 実装上の注意

- パラメータは定数としてまとめて管理されている箇所（`config.py` または `spec.md` 記載の設定ファイル）を探して一元変更すること
- S6戦略の `-10%` 急落閾値（エントリー条件）は **変更しないこと**（仕様書に変更禁止と明記されている）
- 変更後はバックテストを再実行し、Sharpe・最大DDへの影響をログに出力すること

### 3. 毎日の自動実行設定

現在 cron が未設定の可能性があるため、確実に動作する自動実行環境を整備する。

#### 要件

- JST 06:00 にデータ更新（`scripts/update_data.py`）
- JST 06:15 にシグナル生成 + Slack通知（`scripts/daily_signals.py`）
- 実行ログを `logs/` 以下に日付付きで保存すること（例：`logs/2025-06-06.log`）
- 実行失敗時はSlackにエラー通知を送ること

#### 実装方針

cron と GitHub Actions の両方を確認し、プロジェクトの既存構成に合わせて実装すること。
`scripts/setup_cron.sh` が存在する場合はその内容を確認・更新する。

#### cron 設定例（参考）

```cron
0  6  * * 1-5  cd /path/to/stock-trading && uv run python scripts/update_data.py >> logs/$(date +\%Y-\%m-\%d).log 2>&1
15 6  * * 1-5  cd /path/to/stock-trading && uv run python scripts/daily_signals.py >> logs/$(date +\%Y-\%m-\%d).log 2>&1
```

### 4. 2025年以降の成績検証レポート出力

過学習を避けるため、2025年1月1日以降のデータをOut-of-Sample（OOS）として扱い、
実際の成績を定期的に検証・記録する仕組みを追加する。

#### 要件

- `scripts/oos_report.py` を新規作成すること
- 対象期間：2025-01-01 〜 実行日
- 出力内容：

```
=== OOS検証レポート (2025-01-01 〜 2025-06-06) ===

全体
  トレード数   : 38件
  勝率        : 54.2%
  CAGR        : +8.1%
  Sharpe      : 0.81
  最大DD      : -9.3%
  PF          : 2.01

戦略別
  S2: 勝率 53% / CAGR +5.2% / PF 1.88
  S4: 勝率 50% / CAGR +1.9% / PF 1.54
  S6: 勝率 71% / CAGR +1.0% / PF 2.31

バックテスト比較
  Sharpe  バックテスト 0.63 → OOS 0.81  ✅ 改善
  勝率    バックテスト 56.1% → OOS 54.2% ✅ 許容範囲
  最大DD  バックテスト -15.1% → OOS -9.3% ✅ 改善
```

- 毎月1日に自動実行してSlackに投稿すること（cron or GitHub Actions）
- 結果を `reports/oos_YYYY-MM.json` に保存すること
