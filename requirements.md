# 株式投資自動化プロジェクト 要件定義書

**バージョン:** 1.0
**作成日:** 2026-05-16
**ステータス:** ドラフト

---

## 1. プロジェクト概要

### 1.1 目的
過去5年間の日米個別株データから収益機会となるシナリオを抽出し、バックテストで検証されたシナリオに基づき中期スイングで実運用を行い、安定的に超過収益を獲得する。

### 1.2 ゴール
- **定量目標:** 年率リターン 15〜20%、最大ドローダウン -20%以内、シャープレシオ 1.0以上
- **定性目標:** 過学習を避け、実運用で再現可能なシナリオを確立する

### 1.3 スコープ
**対象に含む**
- 日本株（日経平均225構成銘柄）、米国株（S&P500構成銘柄）
- 中期スイング（数週間〜数ヶ月の保有）
- ロングオンリー（信用取引・空売りは初期版では対象外）

**対象に含まない（将来検討）**
- デイトレード、超短期売買
- 信用取引、空売り、オプション、先物
- 自動発注（週末手動発注を前提）
- 仮想通貨、債券、商品

### 1.4 前提条件
| 項目 | 値 |
|------|---|
| 投資元本 | 100〜500万円 |
| 開発時間 | 週5時間以内（週末中心） |
| 運用負荷 | 平日日次5分、週末数時間 |
| 主目的 | 実運用で利益を出す |
| 最優先事項 | バックテスト精度（過学習回避） |

### 1.5 成功基準
**実弾投入の最低基準（必要条件、全て満たす）**
- アウトオブサンプル期間でシャープレシオ ≥ 1.0
- 最大DD ≤ -20%
- ペーパートレード3ヶ月以上で月次プラス継続
- アウトオブサンプル期間で取引回数 ≥ 30回

**理想基準（フル投入の十分条件）**
- アウトオブサンプル期間でシャープレシオ ≥ 1.5
- 最大DD ≤ -15%
- ペーパートレード6ヶ月以上で安定

**段階的投入ルール**
- 最低基準クリア → 元本の20%で運用開始
- 理想基準クリア → 段階的に拡大（最終的に100%）

### 1.6 撤退基準
- プロジェクト開始から24ヶ月経過しても最低基準をクリアできない場合、方法論の根本見直しまたは撤退を判断
- 実運用開始後、6ヶ月連続で月次マイナス継続時は一時停止しレビュー

## 2. 投資戦略

### 2.1 ポートフォリオ構成
- 保有銘柄数：3〜7銘柄
- 1銘柄あたり最大投資比率：20%
- 資金配分初期値：採用シナリオ間で等配分（運用後にバックテスト性能で再調整）

### 2.2 採用シナリオ

| ID | 名前 | タイプ | 想定保有期間 |
|----|------|--------|---------|
| S2 | 52週高値ブレイクアウト | 順張り・トレンドフォロー | 1〜6ヶ月 |
| S3 | 押し目買い（RSI＋トレンド） | 順張り・押し目 | 数週間〜1ヶ月 |
| S4 | 決算後ドリフト（PEAD） | イベントドリブン | 2〜3ヶ月 |
| S6 | 平均回帰（短期反発） | 逆張り・短期 | 数日〜2週間 |

各シナリオの詳細仕様（エントリー条件、エグジット条件、ロスカット、ポジションサイズ）は別ドキュメント「シナリオ仕様書」にて定義する。

### 2.3 シグナル競合時のルール
- 同一銘柄に複数シナリオがシグナルを出した場合、エントリー時系列が早いシナリオを優先
- 既ポジション銘柄に別シナリオのシグナルが出ても重ねて建てない
- ポートフォリオ全体の銘柄数上限（7銘柄）に達した場合、新規シグナルは見送り

### 2.4 マクロイベント時の停止ルール
以下に該当する場合、新規エントリーを一時停止（既存ポジションは継続管理）。
- VIX > 35（米国市場のパニック水準）
- 日経平均が直近20営業日高値から -10% 以上下落
- FOMC、日銀金融政策決定会合の前後1営業日

### 2.5 シナリオ修正トリガー
- 直近20営業日連続でシャープレシオが学習時の50%未満で推移 → 該当シナリオを一時停止しレビュー
- 30日以上のドローダウン継続 → 一時停止
- 修正・再パラメータ調整は四半期に1回を原則とし、過剰修正を避ける

## 3. システム要件

### 3.1 機能要件

**データ取得**
- 日米株価データの日次取得（始値、高値、安値、終値、出来高、調整後終値）
- 過去10年分の履歴データ蓄積（生存者バイアス回避のため上場廃止銘柄も含む）
- 決算データ取得（S4で必須、EPS実績・予想、発表日）
- 為替レート（USD/JPY）取得
- データ品質チェック（欠損、外れ値、整合性）の自動化

**バックテスト**
- 全シナリオの過去データ実行
- ウォークフォワード分析（学習12ヶ月→検証3ヶ月のローリング）
- アウトオブサンプル検証（直近1年は学習対象外）
- 評価指標算出：CAGR、シャープレシオ、最大DD、勝率、プロフィットファクター、平均保有日数
- 対ベンチマーク（日経平均、S&P500、バイ&ホールド）超過リターン算出
- スリッページ・手数料・税金・為替コスト反映

**シグナル生成・運用**
- 日次バッチでシグナル生成（米国市場終了後、日本時間早朝）
- 週末レビュー用レポート自動生成
- ペーパートレードログ記録
- 予実差分の日次ロギング

**ポートフォリオ管理**
- 現在ポジションの管理
- 全体エクスポージャー、銘柄重複、為替エクスポージャーの可視化
- リスク指標（VaR、ベータ等）の算出

**フィードバック**
- 予測 vs 実績の信頼区間付き比較
- シナリオ別パフォーマンス追跡
- シナリオバージョン管理（変更履歴とパフォーマンス紐付け）

### 3.2 非機能要件
- **可用性:** ローカル環境での実行を前提（クラウド不要）
- **再現性:** 同一条件で同一結果が得られること（乱数シード固定）
- **拡張性:** シナリオ追加が容易な設計（プラグイン形式）
- **監査性:** 全シグナル・売買判断のログ保存
- **セキュリティ:** APIキー、口座情報の環境変数管理（Gitに含めない）
- **バージョン管理:** ソースコード・設定・ドキュメントはGit管理（詳細は3.4節）

### 3.3 環境要件
- **言語:** Python 3.11以上
- **DB:** DuckDB（軽量・高速、ローカル運用に最適）
- **データソース（候補、実装時確定）**
  - 日本株：J-Quants API、yfinance、Stooq
  - 米国株：yfinance、Alpha Vantage
  - 決算データ：未定（実装フェーズで調査）
- **可視化:** Streamlit または Jupyter Notebook
- **証券会社:** 楽天証券（手動発注）
- **バージョン管理:** Git（GitHub プライベートリポジトリ）

### 3.4 Git管理方針

**リポジトリ**
- ホスティング：GitHub プライベートリポジトリ
- リモート兼バックアップとして使用

**ブランチ戦略**
- main直接コミット（個人プロジェクト・週5時間制約のためシンプル運用）
- 大規模変更時のみ feature ブランチを切る（任意）

**リポジトリ構成**

```
stock-trading/
├── .gitignore
├── .env.example            # 環境変数テンプレート（実値含めず）
├── README.md
├── requirements.md         # 要件定義書
├── docs/                   # 設計書・シナリオ仕様書・運用手順書
├── src/                    # ソースコード
│   ├── data/               # データ取得・前処理
│   ├── scenarios/          # シナリオロジック
│   ├── backtest/           # バックテスト
│   ├── portfolio/          # ポートフォリオ管理
│   ├── reporting/          # レポート生成
│   └── utils/
├── tests/                  # テストコード
├── notebooks/              # 探索的分析用Jupyter
├── config/                 # 設定ファイル（シナリオパラメータ等YAML）
├── scripts/                # 運用バッチスクリプト
└── pyproject.toml
```

**Git管理対象**
- ソースコード全般（src/, tests/, scripts/）
- 設定ファイル（config/）：シナリオパラメータYAML等
- ドキュメント（docs/, requirements.md, README.md）
- バックテスト結果サマリー（results/summary/）

**Git管理対象外（.gitignoreで除外）**
- 機密情報：`.env`、APIキー、口座情報
- 株価・決算データ本体（容量大、再取得可能）
- ログファイル（logs/）
- DBファイル（`*.duckdb`）
- バックテスト結果生データ（results/raw/）
- Python関連：`__pycache__/`、`.venv/`、`*.pyc`
- IDE関連：`.vscode/`、`.idea/`

**コミットルール**
- Conventional Commits形式を採用
  - `feat:` 機能追加
  - `fix:` バグ修正
  - `docs:` ドキュメント変更
  - `refactor:` リファクタリング
  - `test:` テスト追加・修正
  - `chore:` ビルド・設定変更
- 1コミット = 1論理単位
- 動作確認できないコードはコミットしない

**シナリオのバージョン管理**
- シナリオパラメータは `config/scenarios/*.yaml` でYAML管理
- パラメータ変更時は必ずコミット
- バックテスト結果との紐付けはコミットハッシュで行う
- 採用パラメータ変更時はコミットメッセージに変更理由を明記

## 4. 評価期間設計

### 4.1 期間区切り（初期値、実装時に再調整）

| 期間 | 用途 | 内容 |
|------|------|------|
| 2018/1〜2022/12（5年） | 学習期間 | パラメータ最適化に使用 |
| 2023/1〜2024/12（2年） | 検証期間 | 学習結果の汎化性能確認 |

### 4.2 ウォークフォワード設計
学習12ヶ月 → 検証3ヶ月をスライドし、検証期間全体での累積パフォーマンスを評価する。

## 5. リスク管理

### 5.1 ポジションリスク
- 1銘柄あたり最大20%
- 1セクター集中を避ける（同一セクター最大3銘柄）
- 流動性フィルタ：過去20日平均出来高×売買代金が一定基準以上の銘柄のみ

### 5.2 ポートフォリオリスク
- 最大ドローダウン -20% でサーキットブレーカー作動、全シナリオ一時停止
- 為替エクスポージャー：米国株比率に応じてヘッジ要否を検討（初期版ではノーヘッジ）

### 5.3 オペレーショナルリスク
- APIキー・口座情報は環境変数管理
- バックアップ：Git+クラウドストレージ
- データソース冗長化（無料APIの障害に備え複数候補）

## 6. プロジェクト計画

### 6.1 開発フェーズ

| フェーズ | 内容 | 想定期間 |
|---------|------|---------|
| Phase 0 | 要件定義・設計（本ドキュメント） | 1〜2週間 |
| Phase 1 | データ基盤構築 | 3〜4週間 |
| Phase 2 | シナリオ実装 | 4〜6週間 |
| Phase 3 | バックテスト・評価 | 3〜4週間 |
| Phase 4 | ダッシュボード・運用ツール | 2〜3週間 |
| Phase 5 | ペーパートレード | 3〜6ヶ月 |
| Phase 6 | 実弾投入（段階拡大） | 継続 |

合計開発期間：3〜5ヶ月（週5時間制約下）
実弾投入開始：開発開始から6〜9ヶ月後（最短）

### 6.2 マイルストーン
- M1: データ基盤完成（全銘柄日次データ取得・蓄積）
- M2: 全シナリオのバックテスト完了
- M3: ダッシュボード稼働、ペーパートレード開始
- M4: ペーパートレード3ヶ月クリア、実弾投入20%開始
- M5: ペーパートレード6ヶ月クリア、段階拡大

## 7. 体制・運用

### 7.1 役割
個人プロジェクトのため、開発・運用・意思決定すべて単一担当。

### 7.2 運用ルーティン
- **平日朝（5分）:** 日次シグナル確認、アラート確認
- **平日（必要時）:** ロスカットシグナル発生時のみ対応
- **週末（数時間）:** ポジションレビュー、シナリオパフォーマンスレビュー、必要に応じて再学習

### 7.3 レビューサイクル
- **日次:** シグナル・実績差分
- **週次:** ポートフォリオ全体、ベンチマーク比較
- **月次:** シナリオ別パフォーマンス、修正検討
- **四半期:** パラメータ再調整、シナリオ追加・削除判断
- **年次:** 全体方針見直し

## 8. 用語集

| 用語 | 定義 |
|------|------|
| シナリオ | 売買ルールの集合体（エントリー、エグジット、ロスカット、ポジションサイズを含む） |
| シグナル | シナリオが生成する具体的な売買指示 |
| ポジション | 保有中の銘柄 |
| エクスポージャー | 市場リスクへの暴露量（金額または比率） |
| シャープレシオ | リスク調整後リターン指標 = (リターン - 無リスク金利) / 標準偏差 |
| 最大DD（ドローダウン） | 高値からの最大下落率 |
| ウォークフォワード分析 | 学習期間と検証期間を時系列にスライドさせる検証手法 |
| アウトオブサンプル | 学習に一切使用していない期間のデータ |
| PEAD | Post-Earnings Announcement Drift、決算後の株価ドリフト現象 |
| バックテスト | 過去データでの戦略シミュレーション |
| ペーパートレード | 実弾を使わず仮想的に売買を記録する模擬運用 |
| カーブフィッティング | 過去データに過剰適合し汎化性能を失うこと |
